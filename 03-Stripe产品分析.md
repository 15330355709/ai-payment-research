# Stripe 产品分析 — ACP 协议与超越支付产品线

> 本文分析 Stripe 在 AI 代理支付领域的产品布局，包括 Agentic Commerce Protocol (ACP)、智能体商务套件，以及加密货币、机器支付等"超越支付"产品线。

## 1. Stripe 的入场方式：ACP + 商业套件

- **来源**: [Stripe Blog - Agentic Commerce Suite](https://stripe.com/blog/agentic-commerce-suite)
- **协议**: [Agentic Commerce Protocol (ACP)](https://www.agenticcommerce.dev/)
- **合作方**: OpenAI（首发平台是 ChatGPT）

### 1.1 一句话理解

> Google 的思路是"我来定义 AI 代理支付的标准协议，大家都用我的协议"。
> **Stripe 的思路是"商户已经在用我了，我帮你一键接入 AI 代理渠道，就像多加一个销售渠道"。**
>
> Google 是从协议层往上建生态，Stripe 是从商户层往下接协议。

### 1.2 核心产品：Agentic Commerce Suite

Stripe 的产品定位非常直接——帮助现有商户"一键接入" AI 代理购物：

```
商户现在的问题：

"我想让 ChatGPT/Gemini 这些 AI 代理能卖我的东西，
 但每接一个代理平台就要花 6 个月做对接，太累了。"

Stripe 的方案：

"你的商品目录接到 Stripe，在后台勾选想接入的 AI 代理平台，
 剩下的事（发现、结账、支付、风控）我全帮你搞定。"
```

Agentic Commerce Suite 包含三个核心能力：

| 能力 | 说明 |
|------|------|
| **商品发现** | Stripe 托管 ACP 端点，AI 代理可以搜到你的商品（价格、库存、描述实时同步） |
| **结账流程** | 基于 Stripe Checkout Sessions API，处理税费、运费、库存等，商户保持控制权 |
| **支付风控** | 处理 SharedPaymentToken，配合 Stripe Radar 做代理场景的反欺诈 |

首批接入品牌：Etsy、Urban Outfitters、Coach、Kate Spade、Ashley Furniture、Revolve 等。

### 1.3 核心创新：SharedPaymentToken (SPT)

SPT 是 Stripe 在 ACP 协议中发明的**支付凭证**，相当于 AP2 的 Mandate 在 Stripe 体系中的对应物：

```
传统支付：
  用户把信用卡号给商户 → 商户直接扣款

SPT 支付：
  用户的信用卡已存在 AI 代理平台（如 ChatGPT）
  → 代理生成一个 SPT（不含真实卡号）
  → SPT 发给商户
  → 商户用 SPT 通过 Stripe 完成扣款

  SPT 的限制：
  · 绑定特定商户（不能拿去别的商户用）
  · 限定金额上限
  · 限定有效期
  · 全程可追踪
```

| 对比 | AP2 的 Mandate (VDC) | Stripe 的 SPT |
|------|---------------------|---------------|
| **本质** | 加密签名的意图证明 | 受限的支付 Token |
| **保护什么** | 用户意图不被篡改 | 支付凭证不被滥用 |
| **谁签名** | 用户自己用私钥签 | AI 代理平台生成 |
| **适用范围** | 跨平台、跨支付方式 | Stripe 生态内 |
| **技术基础** | 密码学 + 可验证凭证 | Stripe API + Token 化 |

### 1.5 Stripe 的战略逻辑

**入场方式：从商户侧切入，"卖铲子"策略**

```
Stripe 的优势：
  · 全球数百万商户已经在用 Stripe
  · 商户的商品目录、支付方式、风控系统都在 Stripe 上
  · 商户信任 Stripe

Stripe 的做法：
  "你不需要懂 AI，不需要懂协议，
   只要在 Stripe 后台点几下，你的商品就能被 AI 代理发现和购买。"
```

**与 OpenAI 的深度绑定**：ACP 协议由 Stripe 和 OpenAI 联合开发，ChatGPT 是首个实现 ACP 的平台。ChatGPT Instant Checkout 已在美国上线，接入了 Etsy、Shopify 等商户。

### 2.1 加密货币产品线（Crypto）

Stripe 通过 2025 年 **$11 亿收购 Bridge**（稳定币基础设施公司），构建了从"发行→持有→支付→出金"的完整稳定币闭环。

#### 产品矩阵

```
┌─────────────────────────────────────────────────────────────┐
│                    Stripe Crypto 全景                        │
│                                                             │
│  ① 稳定币支付（面向终端用户）                                  │
│     · 用户用加密钱包付款 → 商户收到美元（自动换汇）              │
│     · 支持 USDC（Ethereum/Solana/Polygon/Base）              │
│     · 支持 USDP、USDG 等                                    │
│     · 集成到现有 Checkout/Elements/Payment Links              │
│     · 单笔限额 $10,000                                      │
│     · 无争议/退单风险（用户已加密签名认证）                     │
│                                                             │
│  ② 稳定币金融账户（面向企业）                                  │
│     · 企业持有稳定币余额（USDC 或 USDB）                      │
│     · 100+ 国家可用                                          │
│     · 银行转账 ↔ 稳定币 双向转换                              │
│     · 发工资/付款给全球供应商                                  │
│     · 托管方：Bridge（Stripe 子公司）                         │
│     · 储备金由 BlackRock 管理                                 │
│                                                             │
│  ③ 稳定币发卡（面向平台）                                     │
│     · 发行虚拟/实体预付卡，余额由稳定币支撑                     │
│     · 持卡人可在任何接受 Visa 的地方消费                       │
│     · 稳定币余额自动转换为法币完成刷卡                          │
│     · 私有预览阶段                                            │
│                                                             │
│  ④ 开放发行平台（面向发币方）                                  │
│     · 任何企业几行代码就能发行自己的稳定币                      │
│     · Bridge 处理储备管理、合规、流动性                         │
│     · 储备由 BlackRock + Fidelity 管理                        │
│     · 首批案例：Phantom 的 CASH、MetaMask 的 mUSD             │
│                                                             │
│  ⑤ 法币→加密入金通道（面向开发者）                              │
│     · Stripe 托管的入金页面 或 嵌入式组件                      │
│     · 用户用信用卡/银行转账买加密货币                           │
│     · 支持 iOS/Android/React Native                          │
└─────────────────────────────────────────────────────────────┘
```

#### 产品逻辑

Stripe 的加密货币策略**不是"让大家炒币"，而是"让稳定币变成更好的美元"**：

```
传统跨境转账：
  美国企业 → 银行 → SWIFT → 中间行 → 本地银行 → 菲律宾员工
  耗时：3-5 天 | 费用：$25-50 | 汇率损失：1-3%

Stripe 稳定币转账：
  美国企业 → Stripe 金融账户（稳定币）→ 链上转账 → 员工钱包
  耗时：秒级 | 费用：近零 | 汇率：实时透明

关键：用户不需要懂"区块链"，只需要在 Stripe 后台操作
```

**USDB 是 Stripe 的秘密武器**：Bridge 发行的 USDB 是"闭环稳定币"，与 USDC 不同——USDB 的储备金利息归 Stripe/Bridge，而不是像 USDC 那样归 Circle。这意味着 Stripe 正在构建自己的"货币层"，逐步减少对 Circle 的依赖。

### 2.2 智能体商务产品线（Agentic Commerce）

这是 Stripe 面向"AI 代理帮用户购物"场景的产品线，核心是让 AI 代理平台（ChatGPT 等）和商户之间能够完成交易。

#### 产品架构

```
┌──────────────────────────────────────────────────────────────┐
│               Stripe Agentic Commerce 架构                    │
│                                                              │
│  ┌──────────┐      ┌────────────────┐      ┌──────────────┐  │
│  │  Buyer   │      │   AI Agent     │      │   Seller     │  │
│  │  (买家)   │      │  (ChatGPT等)   │      │   (商户)     │  │
│  │          │      │               │      │              │  │
│  │ 说"我要   │─────→│ 搜索商品       │─────→│ ACP 端点     │  │
│  │  买鞋"    │      │ 展示选项       │←─────│ 返回商品      │  │
│  │          │      │ 发起结账       │      │              │  │
│  │ 确认购买  │─────→│ 生成 SPT      │─────→│ 用 SPT 扣款  │  │
│  │          │      │              │      │ (通过 Stripe) │  │
│  └──────────┘      └──────┬───────┘      └──────────────┘  │
│                           │                                  │
│                    ┌──────▼───────┐                          │
│                    │Payment       │                          │
│                    │Provider      │                          │
│                    │(Stripe)      │                          │
│                    │· Token 化     │                          │
│                    │· 风控 (Radar) │                          │
│                    │· 结算         │                          │
│                    └──────────────┘                          │
└──────────────────────────────────────────────────────────────┘
```

#### 三个核心子产品

**① Agentic Commerce Protocol (ACP) —— 协议层**

ACP 定义了 AI 代理和商户之间的标准化交互接口：

| 端点 | 作用 |
|------|------|
| `CreateCheckout` | 创建结账会话（商品、数量、客户信息） |
| `UpdateCheckout` | 更新结账（改数量、地址、优惠码等） |
| `CompleteCheckout` | 完成结账（提交 SPT 和支付） |
| `CancelCheckout` | 取消结账 |

支持两种实现方式：RESTful HTTP 和 MCP Server。

**② Agentic Commerce Suite —— 商户工具**

解决的问题：商户接一个 AI 代理平台要 6 个月，Stripe 让它变成一键操作：

```
商户操作：
  1. 把商品目录接入 Stripe（上传或从现有 ERP 同步）
  2. 在 Stripe 后台勾选要接入的 AI 代理平台
  3. 完成 ✓

Stripe 自动处理：
  · 商品信息同步给各 AI 代理平台
  · 结账流程（税费、运费、库存）
  · 支付处理 + 风控
  · 订单事件通知 → 商户用自己的物流系统发货
```

已接入的电商平台：Wix、WooCommerce、BigCommerce、Squarespace、commercetools。

**③ SharedPaymentToken (SPT) —— 支付凭证**

SPT 是整个体系的安全核心，设计非常精巧：

```
SPT 的生命周期：

① 用户在 ChatGPT 上存了一张 Visa 卡

② 用户说"帮我在 Etsy 上买这个花瓶"

③ ChatGPT 通过 Stripe API 生成一个 SPT：
   {
     "id": "spt_1RgaZc...",
     "usage_limits": {
       "currency": "usd",
       "max_amount": 5000,        // 限额 $50.00
       "expires_at": 1751587220   // 30 分钟后过期
     },
     "seller_details": {
       "network_id": "etsy",      // 只能 Etsy 用
       "external_id": "shop_123"  // 只能这个店铺用
     }
     // ❌ 不包含真实卡号
     // ✓ 只含有限信息：卡品牌 + 后四位
   }

④ SPT 发给 Etsy → Etsy 用 SPT 创建 PaymentIntent → Stripe 扣款

⑤ Webhook 通知：
   · Etsy 收到 "spt.used" 事件 → 确认收款
   · ChatGPT 收到 "spt.used" 事件 → 通知用户
   · SPT 使用后自动失效，不能二次使用
```

**SPT vs AP2 Mandate 的核心区别**：

| 维度 | SPT (Stripe) | Mandate/VDC (AP2) |
|------|-------------|-------------------|
| **设计哲学** | "我来管支付安全" | "让密码学保证安全" |
| **信任根基** | Stripe 平台信任（中心化） | 加密签名验证（去中心化） |
| **卡号保护** | Token 化，Stripe 托管真实卡号 | 与卡号无关，是意图证明 |
| **适用场景** | 传统信用卡/借记卡支付 | 任何支付方式（含加密） |
| **商户门槛** | 已用 Stripe 的商户几乎零改动 | 需要深度集成密码学组件 |

### 2.3 机器支付产品线（Machine Payments）

这是 Stripe 面向"机器之间直接付钱"的产品线——没有人类参与，AI 代理或程序自主用加密货币支付。

#### 与智能体商务的区别

```
智能体商务（Agentic Commerce）：
  用户 → "Gemini，帮我买鞋" → AI 代理 → 商户
  ↑ 有人类参与：用户发起、用户确认、用户的信用卡付款

机器支付（Machine Payments）：
  AI 代理 A → 需要调用翻译 API → 直接用 USDC 付费 → 拿到翻译结果
  ↑ 无人类参与：代理自主决定、自主付款（加密货币）、自主拿结果
```

#### 产品特点

| 特性 | 说明 |
|------|------|
| **协议** | 基于 **x402 协议**（Coinbase 开发，Stripe 集成） |
| **最低金额** | 0.01 USDC（约 ¥0.07） |
| **支持链** | Base（以太坊 L2） |
| **Gas 费** | Stripe 代付，商户零成本 |
| **结算方式** | 链上结算后自动进入 Stripe 余额 |
| **报表/退款** | 和普通 Stripe 支付完全一致 |
| **隐私** | 每笔支付生成唯一存款地址，减少链上关联 |

#### 集成方式

Stripe 把 x402 集成做得极其简单——加一行中间件：

```javascript
// 给 /paid 端点加上支付门槛：每次请求收 $0.01 USDC
app.use(
  paymentMiddleware({
    "GET /paid": {
      price: "$0.01",
      network: "base",
      payTo: createPayToAddress,   // Stripe 自动生成存款地址
    }
  })
)
```

背后 Stripe 自动处理：创建 PaymentIntent → 生成链上存款地址 → 等待链上确认 → 自动 capture → 资金进入 Stripe 余额。

#### 与 Coinbase x402 的关系

Stripe 做机器支付时，做了一个出人意料的选择——**直接采用了竞争对手 Coinbase 开发的 x402 协议**：

```
为什么 Stripe 不自己做一套机器支付协议？

① x402 已经有了社区和采用量，重新发明轮子没有意义
② Stripe 的核心竞争力不在协议层，而在"商户体验"层
③ 通过兼容 x402，Stripe 能接入所有已用 x402 的 AI 代理客户端
④ Stripe 的差异化在于：
   · 链上支付自动汇入 Stripe 余额（统一报表）
   · Stripe 代付 Gas 费
   · 每笔支付独立存款地址（隐私保护）
   · 和 Stripe 现有的退款/争议/多币种结算体系打通
```

| 维度 | 直接用 Coinbase x402 | 通过 Stripe 用 x402 |
|------|---------------------|-------------------|
| 收到的钱 | 在链上钱包里（需自己管理） | 在 Stripe 余额里（和信用卡收入统一） |
| Gas 费 | 自己付或用 Coinbase Facilitator | Stripe 代付 |
| 报表 | 自己做链上对账 | Stripe Dashboard 统一查看 |
| 退款 | 自己发链上交易退 | Stripe 退款流程 |
| 合规 | 自己处理 KYT/KYC | Stripe 内置 |

**核心逻辑**：这三条产品线不是独立的，而是一个完整的飞轮：

1. **Crypto** 让商户能接受稳定币 → 为 Machine Payments 打下基础
2. **Agentic Commerce** 让 AI 代理帮用户在商户购物 → 增加 Stripe 交易量
3. **Machine Payments** 让 AI 代理之间直接付费 → 创造全新的交易场景
4. **所有资金都汇入 Stripe Balance** → 商户不需要管理多套账本

Stripe 的底层假设：**未来的"客户"不仅是人类，还包括 AI 代理和自动化程序**。谁能同时服务好人类客户、AI 代理客户和机器客户，谁就占据了支付的未来。
