# 竞争格局与战略分析

> 本文分析 Google、Stripe、Coinbase、Circle 四大玩家在 AI 代理支付领域的角色定位、商业模式和竞合关系。

## 1. Google 和 Coinbase 的角色与商业卡位

### 1.1 先看 AP2 的角色体系

AP2 定义了 6 个角色，每个角色都是一个"卡位机会"：

```
                    ┌──────────┐
                    │   用户    │  真人，发起购物意图
                    └────┬─────┘
                         │
                    ┌────▼─────┐
                    │ User Agent│  AI 代理，代替用户购物
                    │(Shopping  │  （如 Gemini、ChatGPT）
                    │  Agent)   │
                    └────┬─────┘
                         │
          ┌──────────────┼──────────────┐
          │              │              │
   ┌──────▼──────┐ ┌────▼─────┐ ┌──────▼──────────┐
   │ Credentials  │ │ Merchant │ │ Merchant Payment│
   │  Provider    │ │ Endpoint │ │   Processor     │
   │ (管理支付凭证)│ │(商户代理) │ │  (支付处理方)    │
   └──────────────┘ └──────────┘ └────────┬────────┘
                                          │
                                   ┌──────▼──────┐
                                   │ Network &   │
                                   │   Issuer    │
                                   │(支付网络/发卡行)│
                                   └─────────────┘
```

### 1.2 Google 能扮演的角色（至少 5 个）

| 角色 | Google 的对应产品/能力 | 战略价值 |
|------|----------------------|---------|
| **① 协议制定者** | AP2 + A2A + MCP 的设计者和维护者 | 制定游戏规则，掌握标准话语权 |
| **② User Agent（用户代理）** | **Gemini** 作为帮用户购物的 AI 代理 | 掌控用户入口，流量上游 |
| **③ Credentials Provider（凭证提供方）** | **Google Wallet / Google Pay** 管理用户的支付凭证（信用卡、银行卡的 Token） | 掌控支付凭证，成为"数字钱包" |
| **④ Agent Marketplace 运营方** | **Google Cloud AI Agent Marketplace** | 代理的"应用商店"，掌控分发 |
| **⑤ 基础设施提供方** | **Google Cloud** 提供构建 AP2 代理的云服务（ADK 框架、计算资源） | 卖铲子给挖金矿的人 |

更具体地说：

```
一个完整的 AP2 购物场景中，Google 可以占据多少环节？

用户："Gemini，帮我买一双白色跑鞋"
  │
  │ ① Gemini = User Agent（Google 的）
  │
  ├─→ Gemini 在 AI Agent Marketplace 中搜索商户代理
  │     ④ Marketplace = Google Cloud 运营的
  │
  ├─→ Gemini 找到 Nike 的商户代理（运行在 Google Cloud 上）
  │     ⑤ 基础设施 = Google Cloud 提供的
  │
  ├─→ 用户确认购买 → 调用 Google Wallet 提供支付凭证
  │     ③ Credentials Provider = Google Pay/Wallet
  │
  └─→ 整个流程遵循 AP2 协议
        ① 协议 = Google 设计的
```

**Google 的战略意图**：通过制定开放协议，把自己的产品（Gemini、Google Pay、Google Cloud）嵌入到 AI 代理购物的每一个关键环节。协议是开放的，但 Google 在每个角色上都有先发优势。

### 1.3 Coinbase 能扮演的角色（至少 4 个）

#### 在 x402 协议中（核心地盘）

| 角色 | Coinbase 的对应产品/能力 | 战略价值 |
|------|------------------------|---------|
| **① 协议制定者** | x402 的设计者和维护者 | 加密支付标准的定义权 |
| **② Facilitator（清算方）** | **CDP Facilitator** —— 验证支付签名、提交链上结算、覆盖 Gas 费 | 类似传统支付中的"清算所"角色，**每笔交易抽 $0.001** |
| **③ Wallet Provider（钱包提供方）** | **CDP Wallet API** —— 基于 TEE 的安全密钥管理和签名服务 | 掌控密钥 = 掌控资金安全 |
| **④ 交易所/入金通道** | **Coinbase 交易所** —— 用户在此购买 USDC 充值到钱包 | 资金入口 |

#### 在 AP2 协议中（扩展地盘）

| 角色 | Coinbase 的定位 | 战略价值 |
|------|----------------|---------|
| **⑤ Credentials Provider（加密支付凭证）** | 当 AP2 的支付方式是加密货币时，Coinbase Wallet 作为凭证提供方 | 在 AP2 生态中占据加密支付入口 |
| **⑥ 结算基础设施** | AP2 的 x402 扩展底层就是 Coinbase 的 Facilitator | AP2 用加密货币结算时绕不开 Coinbase |

更具体地说：

```
一个完整的 x402 代理间支付中，Coinbase 占据多少环节？

代理 A："帮我生成一张图片"
  │
  ├─→ 代理 B 返回 402："要付 0.05 USDC"
  │
  ├─→ 代理 A 用钱包签名支付
  │     ③ 钱包 = Coinbase CDP Wallet API（密钥在 Coinbase 的 TEE 里）
  │
  ├─→ 代理 B 提交给 Facilitator 验证和结算
  │     ② Facilitator = Coinbase CDP 运营的（每笔 $0.001）
  │     Facilitator 还帮你付了 Gas 费
  │
  ├─→ USDC 链上转账完成
  │     用户的 USDC 大概率从 Coinbase 交易所买的
  │     ④ 交易所 = Coinbase
  │
  └─→ 整个流程遵循 x402 协议
        ① 协议 = Coinbase 设计的
```

### 1.4 商业模式对比

| 维度 | Google 的赚钱方式 | Coinbase 的赚钱方式 |
|------|------------------|-------------------|
| **流量入口** | Gemini 用户量 → 引导购物流量 → 从商户获取佣金或广告费 | USDC 交易量 → 交易所手续费 |
| **平台抽成** | Agent Marketplace 可收取代理上架费或交易分成 | Facilitator 每笔交易 $0.001 |
| **基础设施** | Google Cloud 卖计算资源给商户 | CDP Wallet API 卖密钥管理服务 |
| **支付处理** | Google Pay 可收取支付处理费 | Gas 费代付（成本极低但锁定用户） |
| **数据价值** | 掌握用户购物意图数据（Intent Mandate 过手） | 掌握链上交易数据和钱包余额 |

### 1.5 合作中的竞争关系

Google 和 Coinbase 看似合作（AP2 + x402 打通），但各有**势力范围**：

```
┌─────────────────────────────────────────────────────────────────┐
│                        AI 支付全景                               │
│                                                                 │
│  ┌───────────── Google 的地盘 ──────────────┐                    │
│  │                                          │                    │
│  │  用户 → Gemini (User Agent)              │                    │
│  │       → Google Pay (Credentials Provider)│                    │
│  │       → 商户代理 (Google Cloud 上运行)     │                    │
│  │       → 信用卡/借记卡结算 (传统支付网络)    │                    │
│  │                                          │                    │
│  │  核心资产：用户、流量、云、支付凭证、协议标准  │                   │
│  └───────────────────┬──────────────────────┘                    │
│                      │                                           │
│                 x402 扩展（交汇地带）                              │
│              AP2 需要加密支付时，调用 Coinbase 的基础设施            │
│                      │                                           │
│  ┌───────────── Coinbase 的地盘 ────────────┐                    │
│  │                                          │                    │
│  │  代理 A → 代理 B (Agent-to-Agent)         │                    │
│  │        → CDP Wallet (签名/密钥管理)        │                    │
│  │        → CDP Facilitator (验证/结算)       │                    │
│  │        → 链上结算 (Base/Ethereum/Solana)   │                    │
│  │                                          │                    │
│  │  核心资产：交易所、钱包、清算、链、USDC 生态  │                   │
│  └──────────────────────────────────────────┘                    │
│                                                                 │
│  ⚠️ 潜在冲突点：                                                 │
│  · 如果 Google 未来自建加密钱包/Facilitator → 可替代 Coinbase      │
│  · 如果 Coinbase 未来做 Agent 平台/Marketplace → 可挑战 Google     │
│  · 目前双方各守一块，通过协议互通 → 互利共赢阶段                     │
└─────────────────────────────────────────────────────────────────┘
```

### 1.6 关键洞察

1. **Google 是"协议定义者 + 入口控制者"**：通过制定 AP2/A2A/MCP 三大协议，定义了 AI 代理经济的游戏规则，同时通过 Gemini 和 Google Pay 控制用户入口和支付凭证。赚的是**流量钱和基础设施钱**。

2. **Coinbase 是"加密支付基础设施垄断者"**：在 x402 体系中，Facilitator（清算）、Wallet（密钥）、Exchange（入金）三个环节全部由 Coinbase 占据。虽然协议是开放的（任何人可以做 Facilitator），但目前只有 Coinbase 有生产级的实现。赚的是**每笔交易的微利 + 交易所手续费**。

3. **"开放协议"是战略武器**：两家都把协议做成开放标准——不是因为慈善，而是为了让整个行业采用自己的标准。一旦标准被广泛采用，在标准中占据关键角色的公司就成了"基础设施"，很难被替代。这和 Google 做 Android 开源、Coinbase 做 Base 链的逻辑一样。

4. **当前是"蜜月期"**：Google 需要 Coinbase 的加密支付能力，Coinbase 需要 Google 的用户和流量。但随着 AI 代理支付规模扩大，双方在交汇地带（加密钱包、清算服务）的竞争不可避免。

## 2. 四大玩家全景对比

### 2.1 入场方式对比

| 玩家 | 入场切入点 | 核心策略 | 类比 |
|------|-----------|---------|------|
| **Google** | 协议标准层（AP2/A2A/MCP） | 制定规则，嵌入自家产品 | 做 Android，定标准、控生态 |
| **Stripe** | 商户工具层（ACP + Suite） | 让现有商户零门槛接入 AI 代理 | 做 Shopify 插件，一键开店 |
| **Coinbase** | 清算基础设施层（x402 + CDP） | 做加密支付的"清算所" | 做 Visa 的角色，每笔交易都过手 |
| **Circle** | 货币层（USDC） | 让 USDC 成为代理间默认货币 | 做美联储，发行"AI 世界的美元" |

### 2.2 产品对比

| 维度 | Google AP2 | Stripe ACP | Coinbase x402 | Circle |
|------|-----------|-----------|--------------|--------|
| **支付凭证** | Mandate (VDC) | SharedPaymentToken (SPT) | PaymentPayload（签名） | 无（用别人的协议） |
| **支付方式** | 全部（卡、银行、加密） | 传统支付（40+ 方式） | 仅加密货币 | 仅 USDC |
| **首发合作** | Gemini + 60 家 | OpenAI ChatGPT | Google A2A | Coinbase x402 |
| **商户接入** | 需深度集成 | 一键接入（Stripe 后台） | 加中间件即可 | 提供钱包 API |
| **开源** | 是（Apache-2.0） | 是 | 是（Apache-2.0） | SDK 开源 |

### 2.3 商业模式对比

| 玩家 | 怎么赚钱 | 规模化条件 |
|------|---------|-----------|
| **Google** | 云服务费 + 支付处理费 + Marketplace 分成 + 广告/流量 | AI 代理购物成为主流 |
| **Stripe** | 每笔交易 2.9% + $0.30（传统 Stripe 费率） | 商户通过 AI 代理卖出更多东西 |
| **Coinbase** | Facilitator $0.001/笔 + 交易所手续费 + 钱包服务费 | 代理间加密支付量增长 |
| **Circle** | USDC 储备金利息（2024 年约 $16 亿） | USDC 流通量增长 |

### 2.4 竞合关系图

```
                    ┌─────────────────┐
                    │     Google      │
                    │  AP2 / A2A 协议  │
                    │  Gemini 代理     │
                    └───┬──────┬──────┘
                        │      │
            x402 扩展合作 │      │ 竞争：都想做用户入口
                        │      │ 和支付凭证管理
                        │      │
  ┌─────────────────┐   │      │   ┌─────────────────┐
  │    Coinbase      │←──┘      └──→│     Stripe       │
  │  x402 清算       │              │  ACP + Suite     │
  │  CDP 钱包        │              │  OpenAI 合作      │
  └───────┬─────────┘              └────────┬────────┘
          │                                  │
          │  都用 USDC                        │ 未来可能支持
          │  作为支付代币                      │ 加密支付
          │                                  │
          └──────────┐     ┌─────────────────┘
                     │     │
                ┌────▼─────▼────┐
                │    Circle     │
                │  USDC 发行方   │
                │  钱包即服务    │
                │              │
                │  "不管谁赢，   │
                │   只要用 USDC  │
                │   我就赚"     │
                └──────────────┘

竞争主轴：
  · Google vs Stripe：谁来定义"AI 代理购物"的标准？
  · Coinbase vs Circle：谁控制加密支付的基础设施？
  · Google + Coinbase 联盟 vs Stripe + OpenAI 联盟：阵营对抗
```

### 2.5 关键洞察

1. **Stripe 是最务实的玩家**：不纠结于协议之争，直接解决商户的痛点——"6 个月接入一个 AI 代理太慢了，用我的 Suite 一键搞定"。ChatGPT 的海量用户 + Stripe 的海量商户 = 最快落地的组合。

2. **Circle 是最"躺赢"的玩家**：不做协议、不做工具、不做平台，只做货币。无论 AP2 还是 x402 还是任何新协议，只要代理之间需要用稳定币交易，USDC 就是首选。Circle 靠储备金利息赚钱，USDC 流通量越大利息越多，根本不需要从每笔交易中抽成。

3. **两大阵营正在形成**：
   - **Google + Coinbase 阵营**：AP2 + A2A + x402，偏向开放协议 + 加密支付，更适合复杂的多代理协作场景
   - **Stripe + OpenAI 阵营**：ACP + ChatGPT Instant Checkout，偏向传统支付 + 快速落地，更适合简单的"帮用户买东西"场景

4. **最终可能不是零和博弈**：就像线上支付领域 Visa、Mastercard、PayPal、支付宝共存一样，AI 代理支付也可能形成多协议共存的局面——简单购物用 ACP/Stripe，复杂协作用 AP2/Google，代理间微支付用 x402/Coinbase，而 USDC 在所有加密场景中通用。
